Borrowed Patterns

Agentic AI is being shaped by two borrowed pattern libraries: the practices humans use to produce reliable work, and the cognitive mechanisms humans use to learn.

The Borrowing Has Moved Up the Stack

The idea that AI borrows from human life is not new. Neural networks are the obvious historical example: a computing architecture inspired, however loosely, by the way biological neurons connect and fire. That borrowing happened near the substrate of the system. It shaped how computation happened.

What feels different about the current wave of agentic AI is where the borrowing is happening. We are no longer only borrowing metaphors from biology to describe model internals. We are borrowing from the way humans organise work, pass judgement, learn from mistakes, ask clarifying questions, and improve through feedback. The inspiration has moved up the stack, from computation toward cognition and practice.

That matters because it gives us a useful lens. The patterns emerging around agents are not a grab-bag of clever tricks. They are part of a broader translation project: taking durable human patterns and turning them into system architecture.

The First Library: Human Work Practices

Software engineering is a useful place to start because it has spent decades learning how to make unreliable human work more reliable without pretending humans are perfect. We use tests to make expectations executable. We use code review to add another layer of judgement. We use CI to catch regressions before they reach production. We use approval gates because some changes deserve friction.

Agentic systems are beginning to inherit those habits. Anthropic's public writing on multi-agent research systems describes a lead agent coordinating specialist subagents. Its documentation on evaluation workflows treats quality as something to test against examples and rubrics, not something to vibe-check after the fact. That is not alien to software engineering. It is recognisably the same family of ideas as review, testing, and acceptance criteria.

The important move is the separation between doing and judging. A model can produce an answer, but another process can evaluate whether that answer satisfied the task. The reviewer may be a human, an automated eval, another agent, or some combination of all three. Either way, the system becomes stronger when judgement is made explicit instead of hidden inside a single generation step.

This is where a lot of AI discourse is still too prompt-centric. Better prompts help, but mature software teams do not rely on one perfect instruction to protect production. They build systems around the work: checks, reviewers, tests, observability, rollback paths, and norms about what good looks like. Agentic AI is heading in the same direction.

The Second Library: Human Cognition

The second pattern library is less about professional practice and more about cognition itself. Some emerging agent patterns look less like software process and more like attempts to reconstruct how people learn.

Take the idea of agents reviewing their own traces after the fact, sometimes described with language like dreaming or consolidation. The interesting part is not the branding. It is the mechanism: experience is not only used in the moment; it is replayed, compressed, and turned into better future behaviour. That is closer to a cognitive metaphor than a CI metaphor.

Dialogue is another example. Matt Pocock's Grill Me skill for Claude Code is a small but sharp pattern: before the agent acts, it pushes back with clarifying questions until the user's intent is clearer. That is not just prompt decoration. It is a way of importing a human collaboration habit into the agent loop. People build shared understanding by explaining, questioning, restating, and correcting each other. Agents need versions of that too.

This is why the work-practice and cognition threads should sit side by side. Review, tests, and approval gates are about discipline. Dialogue, consolidation, and feedback are about learning. Agentic systems need both. A system that can act but cannot be reviewed is risky. A system that can be reviewed but cannot learn from the review is brittle.

Evaluation Is Becoming the Centre of Gravity

Once output becomes cheap, evaluation becomes the bottleneck. This is the deeper reason these borrowed patterns matter. AI makes it easy to generate ten drafts, five plans, or a dozen plausible next actions. It does not automatically tell us which one is true, useful, safe, aligned, well-structured, or worth committing to.

That shifts the valuable work upward. The scarce skill is no longer only producing artefacts. It is defining what good means, making that definition inspectable, and building feedback loops that improve the next attempt. In software terms, this is why evals feel closer to tests than to demos. In product terms, it is why review surfaces may matter as much as generation features. In organisational terms, it is why judgement becomes more visible, not less.

The useful pattern is simple:

generate → evaluate → revise → approve → learn

That loop is already familiar in human work. Writers draft and edit. Engineers implement and review. Teams run incidents and post-mortems. Coaches observe and give targeted feedback. Agentic AI is forcing us to encode those loops more deliberately.

What Comes Next

If this lens is right, we should expect more agent design patterns to come from human learning and collaboration. Some will come from software engineering, but not all of them. The broader question becomes: which human patterns are durable enough to translate?

One candidate is learning through teaching. Humans often understand something more deeply when they have to explain it to someone else. It is not hard to imagine agents consolidating task knowledge by teaching another agent, producing a short explanation for a user, or converting an execution trace into a reusable lesson.

Another is social learning. People learn by watching other people work, not only by being corrected directly. Agentic systems may eventually learn more from observing other agents' successes and failures, especially inside bounded domains where the work can be replayed and evaluated.

A third is the post-mortem. Failure should not be a dead end in an agent loop. If an agent takes the wrong path, the system should be able to inspect why: ambiguous instructions, missing context, weak tool choice, overconfidence, bad retrieval, or a poor evaluation rubric. That diagnosis should feed the next attempt.

The fourth is a shared working surface before commit. Coding agents already show the value of diff-first workflows: the agent proposes a change, the human reviews it, and persistence is gated by approval. I do not think that pattern has been generalised widely enough across normal SaaS yet. The next step is applying it to business objects: development plans, invoices, CRM updates, reports, support replies, policy drafts, or performance notes. The agent drafts into the same surface the human uses, but the database commit still waits for review.

That pattern is powerful because it keeps the agent close to the work without making it invisible. The user is not merely chatting beside the product. They are reviewing proposed changes inside the product, in the context where the consequences matter.

The Lens Is the Point

None of this means agentic AI should imitate humans for its own sake. Some human patterns are inefficient, biased, political, or simply irrelevant to machines. The point is not anthropomorphism. The point is selectivity.

The best teams will ask which human practices survived because they solve real coordination, quality, and learning problems. They will ask which cognitive mechanisms are worth approximating because they improve adaptation. Then they will turn those patterns into explicit architecture: evals, rubrics, review surfaces, memory, teaching loops, post-mortems, and approval gates.

That is the useful shift. We are not just making agents that can do more work. We are gradually surrounding them with the same kinds of feedback, judgement, and learning loops that make human work reliable.

The next wave of agentic AI will not only be built from better models. It will be built from better borrowed patterns.
